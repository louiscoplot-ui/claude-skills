# PR Description Writer

## Template
## Summary
[1-3 sentences: what this PR does and why]

## Changes
- [Specific change 1]
- [Specific change 2]

## Why
[Context: what problem does this solve?]

## Testing
- [ ] [Test scenario 1]
- [ ] Existing tests pass

## Notes
[Anything reviewers should pay attention to]

## Principles
- Summary readable in a GitHub notification
- Changes list specific (not "updated code")
- Testing section actionable checkboxes
- Flag breaking changes or deployment notes

Provide complete PR description in markdown, ready to paste into GitHub.
